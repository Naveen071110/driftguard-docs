# Slack Alerts

Configure DriftGuard to send a Slack notification whenever a schema check returns `BREAKING` or `WARNING`.

---

## Step 1: Create a Slack Incoming Webhook

1. Go to [api.slack.com/apps](https://api.slack.com/apps)
2. Click **Create New App â†’ From scratch**
3. Name it `DriftGuard Alerts`, select your workspace
4. In **Incoming Webhooks**, toggle **On**
5. Click **Add New Webhook to Workspace**
6. Choose the channel (e.g. `#data-alerts`)
7. Copy the webhook URL â€” it looks like:
   `https://hooks.slack.com/services/T.../B.../xxx`

---

## Step 2: Register the Webhook with DriftGuard

```bash
curl -s -X POST https://api.driftguard.dev/v1/webhooks \
  -H "Authorization: Bearer $DRIFTGUARD_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Slack data-alerts channel",
    "target_url": "https://hooks.slack.com/services/T.../B.../xxx",
    "event_types": ["BREAKING", "WARNING"],
    "contract_id": null
  }'
```

---
